# Homework 5 Submission Checklist

Before submitting on Canvas, verify every item below.

- The first page of the PDF report shows full name, student ID, and `Homework 5`.
- The PDF report contains a working public GitHub hyperlink to the exact `Homework_5` repository/folder.
- The report is a PDF, not only a notebook or Markdown file.
- `requirements.txt` includes the packages needed to run the code.
- `src/hw5_vit_resnet.py` exists and contains your final Problem 1 implementation.
- `src/hw5_swin.py` exists and contains your final Problem 2 implementation.
- `Homework5_Problem1_ViT_vs_ResNet18.ipynb` is executed if you choose to submit a notebook artifact.
- `Homework5_Problem2_Swin_Finetuning.ipynb` is executed if you choose to submit a notebook artifact.
- `Results_Problem_1/problem1_summary.csv` contains at least four ViT rows and one ResNet-18 row.
- `Results_Problem_1/problem1_history.csv` contains per-epoch metrics for each Problem 1 model.
- `Results_Problem_2/problem2_summary.csv` contains Swin-Tiny, Swin-Small, and scratch Swin results.
- `Results_Problem_2/problem2_history.csv` contains per-epoch metrics for each Problem 2 model.
- The report includes parameter counts, FLOPs, training time, and test accuracy for Problem 1.
- The report includes training time, parameter counts, and test accuracy for Problem 2.
- The report includes discussion of tradeoffs, not only tables and screenshots.
- All figures referenced in the report are present in the repository.
- The code supports a CPU fallback even if the main runs used a GPU.
- Open the GitHub link in a private/incognito browser window and confirm it is publicly accessible.
